OkamiCode `1.0.1-beta` is a continuity, context, and telemetry release. It fixes the parts that matter after the first impressive demo: projects must still exist after an upgrade, providers must receive useful task context, runtime limitations must be explicit, and the cost dashboard must compare subscriptions with observed usage instead of presenting a monthly extrapolation as money already spent.

| Release focus | What changed |
| --- | --- |
| 🧠 **Shared context** | Providers receive bounded conversation and task state, with deterministic compaction and explicit handoff. |
| 🛡️ **Local continuity** | Legacy data and Keychain identities are recovered, with database backup and startup recovery paths. |
| 🧰 **Code workspace** | The composer, project sidebar, activity stream, workspace panels, and response surfaces were refined. |
| 🔌 **Runtime truth** | OpenCode ACP, capability manifests, packaged binary discovery, conformance tests, and lane health landed. |
| 📊 **Honest usage** | Provider telemetry is normalized; observed cost is primary and monthly projection is clearly secondary. |

> [!IMPORTANT]
> This is still an unsigned and non-notarized beta. OkamiCode improves local safety and recovery, but it does not turn third-party coding CLIs into a sandbox. Only grant a runtime access you would also grant when running that CLI directly.

## ✨ Highlights

### Context survives a provider switch

Provider lanes no longer behave like unrelated people who happened to enter the same room. OkamiCode now persists a provider-neutral conversation history and can hand off the current task state when the selected runtime changes.

The context compiler:

- keeps recent relevant turns;
- preserves the current objective, constraints, decisions, files, commands, and unresolved work;
- deduplicates repeated state;
- compacts deterministically when the budget is exceeded;
- separates context occupancy from tokens actually consumed by a turn.

This does **not** merge native provider sessions or pretend every harness has identical semantics. Each runtime still owns its native session; OkamiCode supplies a bounded shared brief around it.

### Your projects do not become collateral damage

The application rename exposed a nasty class of upgrade failure: a new product identity can make an existing local database or Keychain service look empty even when the data is still on disk.

`1.0.1-beta` adds:

- discovery and continued use of the previous application-data location;
- compatibility with the previous Keychain identity;
- automatic database backup before sensitive startup paths;
- recovery behavior that refuses to silently replace a recoverable database with an empty one.

Projects, tasks, conversations, worktrees, local history, and native-session references remain local and are not deleted merely because a CLI is missing or the application identity changed.

### A quieter, more informative Code workspace

The workbench now spends less screen space narrating every internal twitch:

- agent activity is compact by default and expandable for details;
- long execution histories are easier to scan;
- provider, model, state, duration, and available tokens stay attached to each response;
- project colors are more visible;
- projects can be pinned and sorted with clearer controls;
- active work receives purposeful motion;
- completed background work can surface an unread badge;
- project switching displays loading feedback instead of looking frozen;
- workspace panels behave as toggles and close on a second activation;
- composer hierarchy and context status are easier to read.

### OpenCode is a provider, not the landlord

OpenCode is integrated through its official ACP server. The adapter covers lifecycle, readiness, initialization, model discovery, streamed updates, cancellation, errors, and context-occupancy events.

OkamiCode does not route every provider through OpenCode. Claude Code, Codex, Cursor Agent, Antigravity, Grok, MiMo, and MiniMax continue to use their own adapters and native authentication. ACP expands the catalog without taking ownership of the shared thread.

### Runtime capabilities are now evidence-based

Runtime availability is driven by an authoritative manifest and detected capabilities. The packaged application also resolves user-installed binaries from the expected macOS shell locations instead of assuming that Electron inherited an interactive Terminal `PATH`.

Conformance coverage checks the common runtime contract while preserving provider-specific limitations. The UI can now distinguish:

- ready from missing;
- healthy from degraded;
- supported from version-dependent;
- turn-token usage from context occupancy;
- native quota from unavailable quota.

### Cost comparison uses the money you actually consumed

The usage pipeline now normalizes input, cached input, output, reasoning, and total-token fields emitted under different provider schemas. Historical provider activity is included instead of disappearing when older rows lack newer metadata.

The Home dashboard now leads with:

```text
API consumed · observed period
```

The monthly API-equivalent value remains visible as a projection, but it is no longer presented as already-spent money. OpenRouter prices are used only to estimate equivalent API cost; they are not treated as a provider invoice.

## 🔌 Runtime notes

| Runtime | 1.0.1 Beta status | Usage caveat |
| --- | --- | --- |
| Claude Code | Native adapter | Numeric usage when stream-json exposes it |
| Codex | Native app-server | Numeric usage when app-server exposes it |
| Cursor Agent | Native adapter | Token and quota detail depend on the installed version |
| Antigravity | Native + local companion | Per-turn token accounting remains unreliable |
| Grok CLI | Native adapter | Structured usage is recorded when end events expose it |
| MiMo Code | Native adapter | Native quota may remain web-console-only |
| MiniMax `mmx` | Native adapter | Usage depends on the JSON fields exposed by the CLI |
| OpenCode | ACP adapter | ACP context occupancy is not provider turn spend |

Removing an executable marks that runtime unavailable. It does not delete the project, task, conversation, worktree, database, or native-session reference.

## 🔐 Security and recovery

- Encrypted SQLite remains the canonical local store.
- Database backup runs before sensitive migration or recovery paths.
- Existing `safeStorage`/Keychain material remains addressable after the rename.
- Renderer privileges remain isolated behind validated IPC.
- Capability leases, approvals, expiry, and audit records continue to gate privileged work.
- Diagnostic redaction strips credential-shaped values.
- Shared context is bounded and compiled; it is not an unfiltered dump of the local database.

## 📦 Install or upgrade

The attached DMG contains an **unsigned and non-notarized** Apple Silicon application:

1. Download `OkamiCode-v1.0.1-beta-macOS-arm64.dmg`.
2. Quit any running OkamiCode instance.
3. Open the DMG and drag **OkamiCode** to **Applications**.
4. Launch it from Applications.
5. If macOS blocks the first launch, review and allow it under **System Settings → Privacy & Security**.
6. Open **Settings** and re-scan the installed runtimes.

The release also includes `SHA256SUMS.txt` so the downloaded DMG can be verified:

```bash
cd ~/Downloads
shasum -a 256 -c SHA256SUMS.txt
```

Download both files into the same directory. A valid download prints
`OkamiCode-v1.0.1-beta-macOS-arm64.dmg: OK`.

You can also [build from source](https://github.com/OkamiOps/OkamiCode/blob/v1.0.1-beta/README.md#run-from-source).

## ⚠️ Known limitations

- Only macOS Apple Silicon is packaged.
- The application is not signed or notarized.
- Provider behavior can change when an installed CLI updates.
- Some providers do not expose reliable per-turn tokens or native quota.
- MiMo quota may remain visible only in the official web console.
- OpenRouter-equivalent cost is an estimate and may differ from direct-provider billing.
- A short observation window can make monthly projections noisy; OkamiCode recommends at least seven observed days before using one for cancellation decisions.
